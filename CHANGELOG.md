# Changelog

All notable changes to bankstract-cloud. Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Changed

- Auth provider changed from Clerk to Better Auth (owner directive 2026-06-21). Users + sessions live in our own SQLite (`apps/app/data/auth.db`), not an external auth SaaS. Sign-in is Google/GitHub OAuth + email magic-link (Resend); no password/MFA yet. Removes the login-path subprocessor (NDPR/procurement posture), removes the cost ceiling, and keeps the self-host bundle fully self-contained.
- Payments processor changed from Stripe to Paystack. NGN-first pricing (monthly subscription tiers, see PRD.md Â§ Pricing). International USD surface deferred until the first non-Nigerian lead. Owner directive 2026-06-21: Stripe requires Atlas (US LLC + EIN) for Nigerian merchants, the ICP is Nigerian fintechs that need FIRS-compliant NGN invoices, and the owner already holds a Paystack account.
- Free demo cap moved from 10 parses/hour per IP to 50 parses/month per IP (PRD Â§ Pricing).
- Free demo outputs are watermarked (`apps/worker/.../watermark.py`): CSV gets a 4-line `#` comment banner, JSON gets a `_demo` envelope. Paid/test keys and self-host (direct engine) pass through clean. Additive only, no row or value changes. Demo download UI discloses it.

### Added

- Worker Paystack billing: `paystack.py` (HMAC-SHA512 webhook verify + subscription init + overage invoice), `subscriptions.py` (owner-keyed state store + webhook dispatch, deduped by event), `usage.py` + `tiers.py` (overage metering against the tier cap). Endpoints in `routes/billing.py` (subscribe / status / charge-overage / webhook). The Stripe SDK and `STRIPE_*` config are removed.
- Live-key `402 subscription_inactive` gate in `routes/parse.py`: a `bsk_live_` key parses only under an active subscription; test keys parse free.
- Dashboard checkout: `apps/app` Billing page â†’ `POST /api/billing/init` â†’ worker `/v1/admin/billing/subscribe`; `authorization_url` redirect checkout. Subscription status read from the worker (worker is the source of truth; the Paystack secret stays worker-side).
- Annual billing (15% off prepay) alongside monthly. The worker `SubscribeRequest` takes an `interval` (`monthly` | `annual`); `config.plan_for(tier, interval)` selects the Paystack plan code, and the webhook `plan_code -> tier` reverse map covers both intervals so a `subscription.create` on an annual plan still resolves to its tier. No schema change: tier alone drives caps/overage/gate, the interval only picks which plan is charged (overage stays calendar-month metered). Dashboard adds a monthly/annual toggle (`apps/app` Billing); the active-plan card drops the price line since the interval is not persisted (renewal date conveys cadence). New env: `PAYSTACK_PLAN_{STARTER,GROWTH,SCALE}_ANNUAL` (blank = that annual interval returns 503).
- TypeScript SDK `@logickoder/bankstract` (`packages/sdk`): typed wrapper over `/v1` (`parse`, `redact`, `usage`, `banks`) for Node 18+ backends. Zero runtime deps (global `fetch`/`FormData`), dual ESM + CJS, types bundled from `@bankstract/types` so the published package is self-contained. Errors map the worker `ErrorResponse` envelope to a typed `BankstractError` hierarchy (`AuthError`/`SubscriptionInactiveError`/`PayloadTooLargeError`/`UnsupportedStatementError`/`RateLimitError`/`ServerError`/`TimeoutError`); the API key never appears in a thrown error. First published package in the repo (others stay private workspace source). PRD/docs SDK references updated from the placeholder `@bankstract/sdk` to `@logickoder/bankstract`.
- Production deploy bundle `infra-prod/` (distinct from `infra/`, the public worker+demo self-host bundle): full stack of worker + app (dashboard) + demo behind Caddy, plus a nightly `sqlite3 .backup` -> Cloudflare R2 sidecar. Path-routed under one host (`/v1` worker, `/app` dashboard, `/` demo) so Cloudflare's free single-level Universal SSL covers it (no paid cert). New `apps/app/Dockerfile` (full-deps image; entrypoint runs `drizzle-kit push` to create the Better Auth schema on first boot). `apps/app` gains a build-time `basePath` (`APP_BASE_PATH`) so prod serves under `/app` while dev/e2e stay at root; the auth client uses a build-baked absolute `NEXT_PUBLIC_BETTER_AUTH_URL`. Verified end-to-end via docker: images build, app boots + pushes schema + serves `/app/sign-in` 200 + reaches the worker. Marketing + docs are expected on Cloudflare Pages. Also fixed the `infra/` worker `ENV` var (was the ignored `BANKSTRACT_CLOUD_ENV`) and added Sentry passthrough to both bundles.
- E2E checkout coverage (`apps/app` Playwright): an authenticated user reaches the Billing page, sees the tier grid + monthly/annual toggle, and `Subscribe` posts the right `{tier, interval}` and redirects to the returned `authorization_url`. `/api/billing/init` is stubbed at the browser boundary (no real worker/Paystack); the annual toggle's price + interval are asserted. A shared `signInWithMagicLink` helper drives the dev magic-link flow; the dev link log now carries the email so parallel workers each pick out their own link (no race).
- Sentry error reporting for the Next apps (`apps/app`, `apps/demo`), **server-side only**: `instrumentation.ts` (`register` + `onRequestError`) + a per-app `lib/sentry.ts` that mirrors the worker's privacy posture (`tracesSampleRate: 0`, `sendDefaultPii: false`, a `scrubEvent` before-send stripping request body/headers/cookies/query). No browser SDK, so anything a user types or uploads in the browser (the demo's PDF) can never reach Sentry (Directive 1). No build plugin / no source-map upload (`@sentry/cli` build disabled), keeping the Turbopack build clean and self-contained. Off unless `SENTRY_DSN` is set; pinned by a per-app regression test.
- SDK docs page (`apps/docs` `sdk.mdx`): install, quickstart, redaction, options, and the typed-error table for `@logickoder/bankstract`, registered in the docs nav.
- Worker error reporting via Sentry (`observability.py`), off unless `SENTRY_DSN` is set. Hardened for the privacy invariant (Directive 1): `include_local_variables=False` so PDF bytes in stack-frame locals never ship, `max_request_body_size="never"` so the upload body is never captured, `send_default_pii=False` (no cookies/auth header/IP), `traces_sample_rate=0`, plus a `scrub_event` before-send that strips request data/headers/cookies/query. Settings are pinned by a regression test that fails if any is loosened.
- Worker schema is now owned by Alembic migrations (`db.py` `run_migrations`, no ORM; `0001_baseline`), with a self-heal that stamps a pre-Alembic DB at the baseline.
- Overage auto-invoicing: an in-process daily scheduler (`billing_cron.py`, zero new deps) bills each closed cycle's overage and runs a boot catch-up. Idempotent per `(owner, cycle)` via the `overage_invoices` ledger, so daily cadence and worker restarts never double-bill. The cap + overage rate are frozen per cycle in `cycle_tiers` while the cycle is live (migration `0002_overage_billing`), so a closed-cycle invoice is computed from the economics that applied then, never re-priced if `TIERS` changes later. No-op when Paystack is unconfigured (no fake charges). Reads audit success counts only, no PDF payload (Directive 1). Replaces the manual-only `/v1/admin/billing/charge-overage` path (which stays for ad-hoc use). The metering window is the UTC calendar month (deliberate, not the Paystack subscription anchor): the cap resets on the 1st for everyone and the overage invoice does not align with the base-fee renewal date. `/v1/usage` and the cron share that single window definition (`audit.py`) so the customer never sees a figure that disagrees with the invoice. Revisit to subscription-anchored cycles post-revenue.

- Async parse jobs (engine 0.15.0 progress hooks): `POST /v1/parse/jobs` (202) + SSE `GET /v1/parse/jobs/{id}/stream` (progress events, then a final `result` event) + `GET /v1/parse/jobs/{id}` poll + `GET /v1/parse/jobs/{id}/result`. JSON rides the SSE event; CSV/redact bytes (which cannot ride a JSON event) download from `/result`. Bytes stay in memory and are dropped on completion / TTL eviction (Directive 1); the worker stays single-process (in-memory job store). The demo streams a live progress bar; the demo `429` now carries `Retry-After`.
- Signup funnel: a marketing "Sign in" hero CTA and `signup` resolving to the same-origin `/sign-up` (it was falling back to the demo); the demo links back to the landing via a brand lockup and shows a "Get an API key" CTA at the result (gated to the hosted build).
- Brand mark: the "extract" icon (a lifted accent row pulled from a dim stack) as the favicon, apple-icon, and OG mark across all apps, single-sourced in `packages/ui` (`BrandMark`) + `packages/seo`.

### Changed (structure)

- The four Next apps merged into one `apps/web` runtime: marketing `/`, the demo `/demo`, and the dashboard (`/dashboard`, `/sign-in`, `/sign-up`, `/api/*`), to keep RAM low on the 4GB box. Surface logic moved to `packages/marketing` + `packages/demo`, consumed by `apps/web` and the thin extractable `apps/marketing` / `apps/demo` shells (the thin demo ships in the `infra/` bundle). `apps/app` is now `apps/web`; auth is a route group at `/sign-in` + `/sign-up`, not an `/app` path. Docs moved to the same-domain `/docs` path on Cloudflare Pages (Next basePath `/docs`; Caddy proxies `/docs/*`). `infra-prod` collapses to worker + one `web` behind an internal Caddy, fronted by a shared caddy-docker-proxy, deployed by CI (build -> GHCR -> ssh pull + caddy reload).
- Style is enforced by ESLint `@stylistic` (single quotes, no semicolons, double-quote JSX) rather than prettier: `eslint --fix` normalizes the tree and CI rejects drift, so editor format-on-save cannot push files off-idiom.
- `apps/worker/main.py` split into an orchestrator + `routes/` (health, account, keys, billing, parse) + `state.py` (AppState + auth deps) + `responses.py` (shared error envelope). The `_state(request)` helper became a `get_state` dependency.
- `packages/types` now mirrors the full worker `/v1` contract (key, subscription, overage-usage types), consumed by `apps/app`.
